# Staging deploy (API)

Host the NestJS API on Render (or similar) so the mobile app works without your PC on the LAN.

## Prerequisites

- [Neon](https://neon.tech) Postgres ‚Äî run all migrations in `db/migrations/` (including `003_sync_stats.sql`)
- [Upstash](https://upstash.com) Redis ‚Äî for BullMQ email sync + notifications
- Google OAuth client (same as local) ‚Äî add staging redirect URIs if needed
- [Render](https://render.com) account (free tier works for staging)

## Option A ‚Äî Render Blueprint (`render.yaml`)

1. Push repo to GitHub.
2. Render Dashboard ‚Üí **New** ‚Üí **Blueprint** ‚Üí connect repo.
3. Set secret env vars when prompted:

| Variable | Source |
|----------|--------|
| `DATABASE_URL` | Neon connection string |
| `REDIS_URL` | Upstash `rediss://‚Ä¶` URL |
| `ENCRYPTION_MASTER_KEY` | Same 64-char hex as local (or new ‚Äî invalidates encrypted tokens) |
| `GOOGLE_CLIENT_ID` | Google Console |
| `GOOGLE_CLIENT_SECRET` | Google Console |

`JWT_SECRET` can be auto-generated by Render.

**Staging auth (Expo Go):** The app stores a JWT on your phone. If you switch from local API to Render, you get `401 Invalid token` because Render‚Äôs `JWT_SECRET` differs from local. Fix either:

1. **Sync secret (keeps same session):** Copy `JWT_SECRET` from `apps/api/.env` into Render ‚Üí Environment ‚Üí override the auto-generated value ‚Üí redeploy.
2. **Reset session (after `ALLOW_DEV_AUTH=true` on Render):** In the app ‚Üí Settings ‚Üí **Reset session**, or pull to refresh on the dashboard (app auto-clears bad tokens).

Set `ALLOW_DEV_AUTH=true` on Render so staging can issue dev login tokens (not for public production).

**Generate a new shared secret** (local + Render must match):

```cmd
python scripts\generate_jwt_secret.py
```

Update `JWT_SECRET` in `apps/api/.env` and Render, then **Reset session** in the mobile app.

4. Deploy. Note the URL, e.g. `https://returnrider-api.onrender.com`.

### If deploy still uses an old build command

Render Blueprints **do not always auto-update** build settings on an existing service. After pushing an updated `render.yaml`, either:

**Option A ‚Äî Sync Blueprint:** Render Dashboard ‚Üí **Blueprints** ‚Üí `returnrider-api` ‚Üí **Manual sync** (or re-sync from repo).

**Option B ‚Äî Edit the web service directly:** `returnrider-api` ‚Üí **Settings** ‚Üí **Build & Deploy**:

| Field | Value |
|-------|--------|
| Root Directory | `.` (repo root) |
| Build Command | `NPM_CONFIG_PRODUCTION=false npm ci && npm run build -w @returnrider/api` |
| Start Command | `npm run start -w @returnrider/api` |

Add env var `NODE_VERSION` = `20` if the service still picks Node 24.

The API `build` script also uses `npx` to fetch Nest/TypeScript when devDependencies are omitted (production `npm install`).

## Option B ‚Äî Manual Render Web Service

- **Root directory:** `apps/api`
- **Build:** `npm install && npx prisma generate && npm run build`
- **Start:** `npm run start`
- **Health check path:** `/health`

### Mobile `.env`

```env

```

Restart Expo: `npx expo start -c`

### Google OAuth (physical phone)

Expo Go still uses `https://auth.expo.io/@ahmed5145/returnrider` ‚Äî no change unless you switch to a dev build.

### CORS

API uses `origin: true` (all origins) ‚Äî fine for staging. Tighten with `CORS_ORIGINS` before public launch.

### Email sync scheduler

Runs automatically every 15 min via BullMQ. Disable with `EMAIL_SYNC_SCHEDULER_ENABLED=false` if debugging.

## Local vs staging

| | Local | Staging |
|---|-------|---------|
| API URL | `http://LAN_IP:3000/api/v1` | `https://‚Ä¶.onrender.com/api/v1` |
| PC must be on | Yes | No |
| Free tier sleep | No | Render free spins down after idle |

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `nest: not found` on Render | `NODE_ENV=production` skips devDeps ‚Äî use `NPM_CONFIG_PRODUCTION=false` in build (see `render.yaml`) |
| Build fails on Prisma | Ensure `DATABASE_URL` set; `postinstall` runs `prisma generate` |
| 502 on wake | Free tier cold start ‚Äî wait 30‚Äì60s |
| Gmail token exchange fails | `GOOGLE_CLIENT_ID` / `SECRET` match mobile client |
| Sync not running | Check `REDIS_URL`; Upstash must allow BullMQ connections |
